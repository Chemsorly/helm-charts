── Kubernetes Mode Operations ────────────────────────────────────────────────

Agent Jobs are spawned by the DispatchService. Job naming: caa-{workItemId[0:8]}

View active agent jobs:
  kubectl get jobs -l app.kubernetes.io/managed-by=caa-orchestrator -n {{ .Release.Namespace }}

View pending work items (via orchestrator logs):
  kubectl logs deployment/{{ include "coding-agent-automation.fullname" . }}-orchestrator -n {{ .Release.Namespace }} | grep "Pending"

Leader election Lease:
  kubectl get lease -n {{ .Release.Namespace }}

RBAC: The orchestrator ServiceAccount has permissions for:
  - Jobs (create/get/list/watch/delete)
  - Leases (create/get/update)
  - Secrets (create/delete — per-job credential injection)
  - Pods (get/list — startup failure detection)

{{- if .Values.credentialPools.kiro }}
PVC Pool (kiro credentials):
  Configured PVCs: {{ join ", " .Values.credentialPools.kiro }}
  DispatchService claims an available PVC per kiro Job.
  If pool is exhausted, Jobs wait (stay Pending) until a PVC is released.
{{- end }}
{{- end }}

── Rollback Procedure ────────────────────────────────────────────────────────

To export current DB state back to JSON files (enables rollback to pre-DB version):
  kubectl exec deployment/{{ include "coding-agent-automation.fullname" . }}-orchestrator -n {{ .Release.Namespace }} -- \
    dotnet CodingAgentWebUI.dll export-config --output /tmp/pipeline-export

Then copy out:
  kubectl cp {{ .Release.Namespace }}/$(kubectl get pod -l app.kubernetes.io/component=orchestrator -n {{ .Release.Namespace }} -o jsonpath='{.items[0].metadata.name}'):/tmp/pipeline-export ./pipeline-backup
